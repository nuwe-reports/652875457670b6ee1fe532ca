This document brings together the HTTP request for an API to perform the appointment management functionality in a hospital environment that is part of the Accenture-Nuwe Java Back End Developer challenge [accenture-techhub].

The base URL for all API requests is: https://{url}/api

Endpoints

Rooms

  • GET /rooms returns a list of all rooms
  • GET /room/{roomName} returns a room with the specified name
  • POST /room creates the specified room
  • DELETE /room/{roomName} deletes a room with the specified name
  • DELETE /rooms deletes all rooms

Patient

  • GET /patients returns a list of all patients
  • GET /patient/{id} returns a patient with the specified id
  • POST /patient creates a patient
  • DELETE /patient/{id} deletes a patient with the specified id
  • DELETE /patients deletes all patients

Doctors

  • GET /doctors returns a list of all patients
  • GET /doctor/{id} returns a doctor with the specified id
  • POST /doctor creates a doctor
  • DELETE /doctor/{id} deletes a doctor with the specified id
  • DELETE /doctors deletes all doctors

Appointments

  • GET /appointments returns a list with all appointments
  • GET /appointment/{id} returns an appointment with the specified id
  • POST /appointment creates an appointment
  • DELETE /appointment/{id} deletes an appointment with the specified id
  • DELETE /appointments deletes all appointments

Error responses API-specific

  • 404 NOT FOUND
  • 204 NO CONTENT
  • 406 NOT ACCEPTABLE
  • 400 BAD REQUEST

The next steps cover instructions for a local test of the API.

We need to link a MySQL database to our project by changing some line codes on the appliation.properties file to point to the database:

spring.datasource.url=jdbc:mysql://{database IP}:{database port}/{database name}
spring.datasource.username={Database user name}
spring.datasource.password={database password}

On the project path, we execute the following command to install and start the project:

mvn clean install spring-boot:run

If everything is ok you should get on the last lines something similar to:

INFO 40862 --- [main] o.s.b.w.embedded.tomcat.TomcatWebServer: Tomcat started on port(s): 8080 (http) with context path ''
INFO 40862 --- [main] com.example.demo.TechhubApplication: Started TechhubApplication in 5.797 seconds (JVM running for 6.691)

Once our maven-embedded Tomcat server is up and running, you can use your favorite test application for API testing. All examples are made with Postman using https://localhost:8080 as the base URL.

Examples are shown below.

Room endpoints

GET /rooms

Returns all the rooms on the database if there are any

  • No request parameters

Response

        "roomName": "Pharmacy"
    }
]

This status code doesn't return any response body.

This status code doesn't return any response body.

  • Status code: BAD REQUEST

This status code doesn't return any response body.

The appointment time is incorrect (Start and ends at the same time)
